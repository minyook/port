프로젝트 목표(300자내외)
본 프로젝트는 Overnight.AI라는 서비스명을 통해, 밤새 발표를 준비하는 학생과 직장인을 위한 AI 멘토를 제공하는 것을 목표로 한다. 기존 STT 중심의 평가 한계를 넘어, 컴퓨터 비전 기술을 접목한 멀티모달 발표 평가 시스템을 구축한다. 이를 통해 발표자의 언어적 요소뿐만 아니라 비언어적 요소를 통합 분석하여, 정량적이고 객관적인 평가 점수와 맞춤형 피드백을 제공함으로써 사용자의 발표 역량을 획기적으로 향상시키고자 한다.
내용(500자내외)
  웹 기반의 사용자 UI와 Python 기반 멀티모달 AI 분석 서버를 구축하였다. 사용자가 영상을 업로드하면 Whisper ASR로 음성을 텍스트로 변환하고, Praat 라이브러리를 통해 목소리의 운율과 발화 속도(CPS)를 분석한다. 비전 처리는 YOLOv8과 MediaPipe를 체이닝하여 얼굴 랜드마크 기반 정밀 시선 추적을 수행하며,YOLOv8-Pose로 전신 관절을 추적해 유의미한 발표 제스처만 필터링한다. 또한 컴퓨터 비전 기술을 활용해 슬라이드 원본과 영상의 일치율을 자동 검증한다.이렇게 추출된 멀티모달 시계열 데이터는 타임스탬프 기준으로 융합되어 파인튜닝된 로컬 LLM 기반 채점 엔진에 입력된다. 최종적으로 맞춤형 루브릭에 따른 항목별 점수와 행동 교정 솔루션이 생성되며, 이는 대시보드 시각화 및 PDF 리포트로 변환되어 사용자에게 제공된다. 
기대효과(200자내외)발표자는 시공간의 제약 없이 무제한으로 연습하고 객관적인 AI 피드백을 받아 스스로 약점을 보완할 수 있다. 평가자는 반복적인 채점 업무로 인한 피로도를 줄이고, 데이터에 기반한 공정한 평가를 수행할 수 있다. 궁극적으로 에듀테크 시장에서 경량화된 AI 채점 솔루션으로서 상용화 가능성을 확보한다.

3) 프로젝트 필요성 및 의도최근 교육청 등에서 AI 기반 논술/평가 시스템 도입이 확대되고 있는 추세이다. 하지만 기존 상용 서비스들은 대부분 영어 발음 교정이나 텍스트 문법 분석에 치우쳐 있다. 우리 팀은 비언어적 요소까지 통합 평가할 수 있는 시스템이 필수적이라고 판단하였다. 이를 통해 교육 현장의 공정성을 확보하고 효율적인 발표 훈련 도구를 제공하고자 한다.2. 프로젝트의 목표(1) 공학적 목표Ÿ멀티모달 통합 데이터 파이프라인 및 실시간 최적화 구조 설계 : 고가의 GPU 서버 없이 일반 CPU 환경에서도 초당 30프레임 이상의 비전 추론과 오디오 처리를 병행하기 위해, 백엔드 내 비동기 실시간 큐 구조를 설계함. 이를 통해 비전, 음성, 슬라이드 데이터를 시간축 기준으로 정렬하여 결합하는 멀티모달 정합기술을 확보했다.Ÿ경량 모델 체이닝을 통한 고정밀 안면 및 시선 추적 : 원거리 발표 환경의 한계를 극복하기 위해 YOLOv8 Nano(얼굴 검출)와 MediaPipe Face Mesh(468개 키포인트)를 체이닝하여 인식 성공률을 95% 이상으로 끌어올림. 홍채 중심점과 눈꼬리 간의 변위 벡터를 정량화하고, 시선 편위 감지 알고리즘을 통해 80% 이상의 정면 응시율을 목표로 하는 객관적 지표 계측시스템을 구축함.Ÿ포즈 에스티메이션 기반의 행동 인식 및 노이즈 필터링 : YOLOv8-Pose를 활용해 신체 33개 관절을 추적하며, 단순 움직임과 유효한 제스처를 구분하기 위한 공간 좌표 기반 예외 처리 로직을 직접 구현함. 이를 통해 옷을 만지거나 머리를 긁는 등의 노이즈를 90% 이상 차단하고 유효한 지시 동작만을 시계열 데이터로 적재함.Ÿ운율 및 발화 지표 정밀 계측 엔진 구현 : Whisper ASR을 통한 고정밀 STT뿐만 아니라, Praat 라이브러리를 연동하여 음성의 물리적 특성(Jitter, Shimmer)과 무음 구간을 분석함. 특히 발표 전달력의 핵심인 CPS 연산 루틴을 구현하여 실시간으로 발화 속도의 적정성(5~7.5 CPS)을 판정하는 기술적 근거를 마련함.ŸPPT 슬라이드 정합 및 LLM 기반 맞춤형 코칭 엔진 : ImageHash 및 python-pptx를 융합하여 영상 내 슬라이드 전환을 85% 이상의 정확도로 자동 감지하고 원본 PPT와 매칭함. 수집된 모든 멀티모달 데이터를 LoRA로 미세조정된 로컬 LLM(Gemma/Llama) 에 입력하여, 단순 점수 산출을 넘어 타임스탬프와 연동된 구체적인 액션 아이템을 생성하는 지능형 피드백 인프라를 완성함.

합성
고려사항영상(Vision), 음성(Audio), 언어(Text), 발표 자료(PPT) 등 서로 다른 형태의 데이터를 하나의 발표 평가 결과로 통합하기 위해서는 각 데이터의 시간 정보와 분석 결과를 정렬하는 기술이 필요하다. 발표 영상에서는 프레임 단위의 시각 정보가 추출되고, 음성 분석에서는 발화 구간 단위의 텍스트와 음성 특징이 생성되므로, 서로 다른 분석 결과를 하나의 시간축 기준으로 결합하는 구조가 필요하였다.수행결과FFmpeg를 이용해 발표 영상에서 오디오와 프레임을 분리하고, Whisper STT의 구간 단위 타임스탬프와 YOLOv8-Pose/MediaPipe의 프레임별 시각 분석 데이터를 시간축 기준으로 정렬하는 파이프라인을 구축하였다. 이를 통해 특정 발표 구간에서 사용자가 어떤 발화를 했는지, 동시에 시선·표정·제스처가 어떤 상태였는지를 함께 분석할 수 있도록 하였다. 분석 결과는 data_combiner.py를 통해 음성 구간과 시각 분석 데이터를 융합하는 형태로 처리하였다.

고려사항발표 평가의 신뢰도를 높이기 위해서는 발표자의 음성, 시선, 표정, 제스처, 자세 등 다양한 요소를 분석할 수 있는 AI 및 컴퓨터비전 기반 분석 엔진이 필요하다. 또한 분석 결과를 단순 수치로 나열하는 것이 아니라, 발표 평가 기준에 맞추어 점수화할 수 있는 루브릭 체계가 요구되었다.
수행결과
YOLOv8-Pose를 활용하여 발표자의 자세와 제스처를 분석하고, MediaPipe Face Landmarker를 활용하여 시선과 표정 정보를 분석하였다. 음성 분석에서는 Whisper를 활용하여 발표 음성을 텍스트로 변환하고, Praat-parselmouth와 자체 규칙 기반 분석을 통해 음성 운율, 필러 표현, 무음 구간, 반복 표현, 발화 속도 등을 분석하였다.이러한 분석 결과를 바탕으로 태도 20점, 음성 30점, 내용 50점의 100점 만점 루브릭 기반 평가 체계를 설계하였다. 태도 영역은 시선 집중도와 제스처/모션을 중심으로 평가하고, 음성 영역은 음성 안정도, 스피치 평정심, 필러 제어를 중심으로 평가한다. 내용 영역은 발화 흐름, 발표 자료 구성, PPT 보조 분석 결과 등을 바탕으로 평가하도록 구성하였다.
제작 및 시험
고려사항사용자가 쉽게 발표 영상을 제출하고 분석 결과를 확인할 수 있는 사용자 친화적인 대시보드와, 영상 분석처럼 시간이 오래 걸리는 작업을 안정적으로 처리할 수 있는 백그라운드 분석 구조가 필요하였다.
수행결과
React 기반 프론트엔드를 구현하여 발표 영상 및 PPT 업로드, 브라우저 발표 녹화, 분석 상태 확인, 결과 대시보드 조회 기능을 제공하였다. 분석 결과는 Recharts 기반 차트, 점수 카드, 마크다운 피드백, 비디오 타임라인 피드백 형태로 시각화하였다.백엔드는 FastAPI 기반으로 구현하였으며, 업로드된 영상에 대해 백그라운드 분석 작업을 실행하도록 구성하였다. 분석 과정에서는 오디오 추출, 프레임 추출, 시각 분석, 음성 분석, 데이터 결합, AI 피드백 생성, 결과 JSON 저장이 순차적으로 수행된다. 이를 통해 사용자가 파일 업로드 후 분석 상태를 확인하고, 완료 시 결과를 조회할 수 있는 구조를 구현하였다.

안정성 및 신뢰성
제한조건영상 분석(YOLO, MediaPipe)과 음성 전처리(Whisper, Praat)를 함께 수행할 경우, 분석 시간이 길어지고 시스템 리소스 사용량이 증가할 수 있다. 특히 대용량 발표 영상 파일을 처리할 때 사용자가 요청 결과를 바로 받을 수 없으므로, 분석 상태를 확인할 수 있는 구조가 필요하였다.해결방안FastAPI의 백그라운드 작업 구조를 활용하여 영상 업로드 요청과 실제 분석 작업을 분리하였다. 시스템은 업로드 이후 작업 ID(job_id)를 생성하고, 사용자는 GET /api/status/{job_id} API를 통해 분석 상태를 확인할 수 있도록 구현하였다. 분석 과정은 오디오 추출, 프레임 추출, 시각 분석, 음성 분석, 멀티모달 데이터 결합, AI 피드백 생성, 결과 저장의 흐름으로 안정적으로 수행되도록 설계하였다.
미학제한조건딥러닝 분석 결과와 음성 지표를 단순 텍스트로만 나열할 경우 사용자가 결과를 직관적으로 이해하기 어렵다. 발표자가 자신의 강점과 약점을 빠르게 파악하기 위해서는 점수, 차트, 피드백을 시각적으로 구성하는 UI가 필요하였다.해결방안React 기반 대시보드 UI를 제작하여 총점, 영역별 점수, 세부 항목 점수, AI 피드백, 타임라인 피드백을 구분하여 표시하였다. Recharts를 활용해 Radar Chart와 Pie Chart 형태로 점수를 시각화하였으며, 마크다운 렌더링을 통해 AI 피드백이 보고서 형태로 가독성 있게 출력되도록 구성하였다. 또한 PDF/Excel 리포트 다운로드 기능을 제공하여 사용자가 결과를 문서로 보관할 수 있도록 하였다.

보완 요청 사항
최종 결과 발표 과정에서 지도교수로부터 발표 영상을 업로드하여 분석하는 방식뿐만 아니라, 사용자가 웹캠을 통해 발표를 실시간으로 촬영하면서 분석할 수 있으면 활용성이 더 높아질 것이라는 피드백을 받았다. 현재 시스템은 발표 영상을 업로드하거나 녹화한 뒤 분석하는 사후 분석 방식에 가깝기 때문에, 발표 중 실시간으로 시선, 제스처, 음성 상태를 분석하고 즉시 피드백을 제공하는 기능은 향후 보완 과제로 도출되었다.또한 분석 결과가 웹 화면에서만 제공될 경우 사용자가 발표 연습 결과를 장기적으로 보관하거나 비교하기 어렵다는 점을 고려하여, 결과를 파일 형태로 저장할 수 있는 기능이 필요하다고 판단하였다. AI 피드백 생성의 경우 로컬 모델 실행 환경이나 API 상태에 따라 응답 속도와 안정성이 달라질 수 있으므로, 상황에 따라 AI 모델을 선택적으로 사용할 수 있는 구조도 필요하였다.개선 결과분석 결과를 문서 형태로 저장할 수 있도록 jsPDF 및 XLSX 라이브러리를 연동한 PDF/Excel 리포트 내보내기 기능을 구현하였다. 이를 통해 사용자는 발표 분석 결과를 파일로 저장하고, 이후 발표 연습 결과와 비교할 수 있게 되었다.또한 로컬 Ollama 기반 Gemma 모델과 Google Gemini API를 선택적으로 사용할 수 있는 구조를 마련하여, 개발 및 시연 환경에 따라 AI 피드백 생성 방식을 조정할 수 있도록 하였다.향후 개선 방향지도교수 피드백을 반영하여 향후에는 업로드 기반 분석뿐만 아니라 웹캠을 이용한 실시간 발표 촬영 및 실시간 피드백 기능을 추가할 필요가 있다. 이를 위해서는 브라우저에서 영상과 음성을 실시간으로 수집하고, 일정 시간 단위로 프레임과 오디오를 분석하여 사용자에게 즉시 피드백을 제공하는 구조가 필요하다. 해당 기능이 구현된다면 발표자가 발표 도중 시선 이탈, 제스처 부족, 말 속도 문제 등을 즉시 확인할 수 있어 발표 연습 효과를 더욱 높일 수 있을 것으로 기대된다.
